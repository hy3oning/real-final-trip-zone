Rules

구조 규칙

  • 엔티티는 단수형 사용
  • DTO는 DTO 접미사 사용
  • 서비스 구현체는 Impl 접미사 사용
  • 문의 도메인은 InquiryRoom / InquiryMessage 기준으로 통일
  • 오타와 중복 명칭은 개정 구조 기준으로 통일

Minimum Backend

최소 백엔드 고정점

반드시 고정할 것

  • 루트 패키지명 com.kh.trip
  • 도메인명과 컨트롤러명
  • 문의 모델 InquiryRoom / InquiryMessage
  • 예약 / 결제 / 리뷰 핵심 엔티티명

팀원 백엔드 범위

  • 실제 엔티티와 repository 구현
  • 상태 전이와 정합성 검증
  • 예약 중복 방지 로직
  • 쿠폰/문의/감사로그 처리
