Overview

프로젝트 개요

프로젝트명: TripZone 유형: 숙소 예약 플랫폼 목표: 설계 기준 고정

TripZone은 사용자가 숙소를 탐색하고 예약, 결제, 리뷰, 문의까지 처리할 수 있는 통합 서비스다. 이번 문서는 현재 구현 데모와 분리해서, 설계 기준만을 제출용으로 정리한 요구사항 기준서다.

이 문서는 com.kh.trip 루트 패키지 기준과 최신 설계 문서를 바탕으로 작성했다.

Requirements

핵심 요구사항

회원

  • 회원은 이메일 기준으로 고유해야 한다.
  • 회원은 일반, 판매자, 관리자 권한을 가질 수 있어야 한다.
  • 로그인 방식은 일반 로그인과 소셜 로그인 확장 가능 구조여야 한다.

판매자

  • 판매자는 일반 회원 확장 개념이다.
  • 판매자는 승인 전까지 숙소 운영 권한이 없다.
  • 판매자 상태는 승인 대기, 승인, 반려, 중지로 관리한다.

숙소/객실

  • 판매자는 숙소와 객실을 등록할 수 있어야 한다.
  • 숙소와 객실은 운영 상태를 가져야 한다.
  • 숙소 상세에는 이미지, 설명, 체크인/체크아웃 정보가 포함되어야 한다.

예약/결제

  • 예약은 객실 단위로 생성한다.
  • 예약 시 숙박일, 인원수, 가격, 할인, 총 결제금액이 저장되어야 한다.
  • 결제 상태는 준비, 결제완료, 실패, 취소, 부분취소, 환불을 지원한다.

리뷰 / 찜 / 문의 / 혜택

  • 리뷰는 숙박 완료 예약에 대해서만 작성 가능해야 한다.
  • 동일 회원이 동일 숙소를 중복 찜할 수 없어야 한다.
  • 문의는 채팅형 구조를 기준으로 한다.
  • 회원은 보유 쿠폰과 마일리지를 예약 시 사용할 수 있어야 한다.

Non-functional

비기능 요구사항

운영성

  • 주요 상태 변경은 추적 가능해야 한다.
  • 삭제는 기본적으로 상태 변경 또는 soft-delete 기반으로 처리한다.

정합성

  • 데이터 정합성은 DB 제약조건과 서비스 검증을 함께 사용한다.
  • 운영용 조회 성능을 위해 주요 검색 조건에 인덱스가 필요하다.
