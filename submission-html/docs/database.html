DB 기준

실행용 DDL은 tripzone-ddl-v2.sql을 따르고, 이 문서는 제출용 요약본이다.

설계 원칙 주요 테이블 핵심 관계 상태값 기준 정합성 기준

Principles

DB 설계 원칙

  • 기본키는 NUMBER(10) + 시퀀스 사용
  • 기본 등록/수정일 컬럼은 REG_DATE, UPD_DATE
  • 상태 기반 운영을 우선하고, 필요 시 soft-delete 컬럼 사용

Integrity

정합성 기준

  • 이메일 유니크
  • 동일 회원/숙소 찜 중복 금지
  • 동일 예약 리뷰 중복 금지
  • 체크아웃 날짜는 체크인 날짜보다 뒤여야 함
  • 환불 금액은 결제 금액 초과 불가
