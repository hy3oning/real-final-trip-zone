TripZone Presentation
01 개요

Project Overview

TripZone는 설계 문서와 구현 기준을 함께 정리한 팀 프로젝트입니다

요구사항, 구조, DB, 문의 모델을 한 흐름으로 정리했다.

com.kh.trip 3-role platform InquiryRoom / InquiryMessage

Project Set

설계 문서 세트

4 + 1
명세서 HTML 4종 발표용 deck 1종

Background

문서와 구현 기준을 같은 흐름으로 맞출 필요가 있었다

  • 문서 기준 정리
  • 상태값 표현 통일
  • 팀 공유용 자료 구성

Approach

핵심 설계 문서와 구현 기준을 분명하게 나눠 정리했다

  • HTML 허브 재구성
  • 통합 기준서, DDL, 메모 연결
  • 프론트와 백엔드 기준 분리

Scope

TripZone의 제품 범위

사용자, 판매자, 관리자 3개 역할 기준으로 설계했다.

User

회원가입, 로그인, 숙소 탐색, 예약, 결제, 리뷰, 찜, 문의, 혜택 사용

Seller

판매자 승인 신청, 숙소/객실 등록, 예약 처리, 문의 응답

Admin

회원/판매자 관리, 이벤트/쿠폰 관리, 문의 모니터링, 감사 로그 관리

Roles

핵심 구조는 3개 역할 분리다

일반 회원

경험 중심 사용자

  • 검색
  • 예약
  • 결제
  • 리뷰
  • 문의

판매자

운영 중심 사용자

  • 숙소 등록
  • 객실 관리
  • 예약 처리
  • 문의 응답

관리자

통제 중심 사용자

  • 판매자 승인
  • 이벤트/쿠폰
  • 문의 모니터링
  • 감사 로그

Structure

루트 패키지명과 레이어 기준을 함께 본다

com.kh.trip.domain com.kh.trip.dto com.kh.trip.repository com.kh.trip.service com.kh.trip.controller

패키지와 레이어 기준을 중심으로 구조를 설명한다.

주요 도메인

  • User / HostProfile / MemberGrade
  • Lodging / Room / Wishlist
  • Booking / Payment / Review

문의 도메인

  • InquiryRoom
  • InquiryMessage
  • InquiryService / InquiryController

Database

DB는 상태 기반 운영 모델로 보강했다

회원

ACTIVE, DORMANT, BLOCKED, DELETED

판매자

PENDING, APPROVED, REJECTED, SUSPENDED

예약

PENDING, CONFIRMED, CANCELED, COMPLETED, NO_SHOW

핵심 테이블

  • USERS, HOST_PROFILES
  • LODGINGS, ROOMS
  • BOOKINGS, PAYMENTS
  • REVIEWS, WISHLISTS

운영 / 혜택 / 문의

  • EVENTS, COUPONS, USER_COUPONS
  • MILEAGE_HISTORY, AUDIT_LOGS
  • INQUIRY_ROOMS, INQUIRY_MESSAGES

Conflict Fix

문의 모델은 메시지 기반 구조로 정리했다

문의방과 메시지를 분리한 구조를 사용한다.

구조InquiryRoom = 문의방
메시지InquiryMessage = 대화 단위
확장읽음 처리 · 상태 관리

Frontend First

이번 구현은 프론트 산출물 중심이다

  • 명세서 HTML 4종
  • 발표용 HTML 1종
  • 인쇄 / PDF 대응

Backend Minimum

백엔드는 연결 기준만 정리한다

  • 패키지와 레이어
  • 도메인명과 상태값
  • 문의 구조와 핵심 엔티티
  • 실제 구현은 팀원 범위

Submission

최종 제출물 구성

문서형

  • 요구사항 명세서
  • 기능 명세서
  • 구조 명세서
  • DB 명세서

기준형

  • DDL
  • 수정 사유 코멘트
  • 발표용 HTML deck

Closing

이번 정리는 설계 문서와 구현 기준을 팀 단위로 공유하기 위한 작업이다

설계 문서와 구현 기준을 같은 흐름으로 참고할 수 있게 정리했다.

Key Result

  • 문서 허브 정리
  • 문의 모델 정리
  • DB 상태 모델 정리
  • 공유용 자료 구성
