수정 코멘트

문서 정리 방향, 유지 기준, 구현 시 참고할 메모를 정리한 페이지다.

Why Changed

주요 정리 항목

01
문서 기준 정리

TripZone 문맥에 맞게 문서 표현과 구조를 다시 정리했다.

02
권한 구조 통일

USER_ROLES, USER_AUTH_PROVIDERS, USER_REFRESH_TOKENS 기준으로 표현을 맞췄다.

03
문의 모델 정리

INQUIRY_ROOMS + INQUIRY_MESSAGES 기준으로 문의 구조를 일관되게 맞췄다.

04
운영 상태 확장

판매자와 회원 운영 상태를 서비스 흐름에 맞게 보완했다.

05
예약 / 결제 / 리뷰 기준 보완

취소 추적, 환불 제약, 리뷰 상태, 중복 방지 기준을 함께 정리했다.

06
운영 로그 범위 추가

승인, 차단, 취소, 쿠폰 발급 같은 운영 행위를 추적할 수 있게 범위를 넓혔다.

Kept

그대로 유지한 것

  • *_NO 기본키 명명
  • REG_DATE, UPD_DATE 중심 시간 컬럼
  • MEMBER_GRADES, USERS, HOST_PROFILES, LODGINGS, ROOMS, BOOKINGS, PAYMENTS 같은 핵심 테이블명

Service Layer

서비스 레이어에서 막아야 하는 것

  • 관리자 권한 사용자의 실제 승인 처리 검증
  • 중복 예약 방지
  • 숙박 완료 예약만 리뷰 작성 가능하도록 검증
  • 회원 쿠폰 재사용 방지
  • 문의방 접근 권한 검증

Impact

이 문서를 볼 때 확인할 것

유지한 축

  • 핵심 테이블명
  • 시간 컬럼 규칙
  • 기본키 명명 규칙

조정한 축

  • 문의 모델
  • 권한 구조
  • 운영 상태값
