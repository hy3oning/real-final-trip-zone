대시보드 요구사항 기능 구조 DB 통합 기준 발표자료

연동 가이드

프론트 화면이 실제 API에서 기대하는 응답 형태를 정리한 문서다.

문서 목적 공통 계약 내 예약 쿠폰 마일리지 결제 내역 문의센터 연결 순서 예시 응답

함께 참고

기능 명세서 통합 기준서 DDL 기준

Purpose

문서 목적

API 응답 기준 마이페이지 우선 요약값 포함 fallback 고려

Oracle 연결이 먼저 성공해도 응답 필드가 최소 수준이면 화면은 단출하게 보일 수 있다. 이 문서는 프론트가 실제로 그리는 데 필요한 응답 구조를 정리한 연동 기준이다.

엔티티를 그대로 노출하기보다 화면 단위 응답을 내려주는 쪽이 안전하다.

Contract

공통 응답 계약

공통 필드

  • id
  • status
  • statusLabel
  • createdAt / updatedAt
  • actions

상단 요약

  • summary.totalCount
  • summary.balance
  • summary.amountTotal
  • summary.activeCount

Bookings

내 예약

구분필드
요약totalCount, upcomingCount, completedCount, canceledCount
기본bookingId, bookingStatus, bookingStatusLabel, checkInDate, checkOutDate, guestCount
숙소lodgingId, lodgingName, roomId, roomName, region, thumbnailUrl
액션canCancel, canReview, canViewPayment

Coupons

쿠폰

구분필드
요약availableCount, expiringSoonCount, usedCount
기본userCouponId, couponName, couponType, discountValue, discountLabel
상태status, statusLabel, issuedAt, expiredAt, isUsable
대상appliesTo

Mileage

마일리지

구분필드
요약balance, earnedThisMonth, usedThisMonth
기록historyId, historyType, historyTypeLabel, title, description, amount, occurredAt

Payments

결제 내역

구분필드
요약paymentCount, paidAmountTotal, refundAmountTotal
기본paymentId, bookingId, paymentStatus, paymentStatusLabel
숙소lodgingName, roomName
금액paymentAmount, refundAmount
수단paymentMethod, paymentMethodLabel, paidAt

Inquiries

문의센터

구분필드
요약openCount, answeredCount, closedCount
기본inquiryRoomId, title, inquiryType, inquiryTypeLabel
상태roomStatus, roomStatusLabel
참조lodgingId, lodgingName, bookingId
최근 메시지latestMessagePreview, latestMessageAt

Priority

연결 우선순위

01
내 예약

summary, 상태, 숙소 정보, 버튼용 id가 가장 먼저 필요하다.

02
쿠폰 / 마일리지

count, balance, 상태 라벨이 화면 밀도에 큰 영향을 준다.

03
결제 내역

결제수단, 금액, 환불 상태까지 같이 내려와야 한다.

04
문의센터

문의방 정보와 최근 메시지 preview가 같이 필요하다.
