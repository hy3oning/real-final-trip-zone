# TripZone 구조 명세서

제출 기준 구조 명세는 `tripzone-structure-spec-v2.md`를 요약한 문서다.

## 1. 패키지 구조

- `com.kh.trip.domain`
- `com.kh.trip.dto`
- `com.kh.trip.repository`
- `com.kh.trip.service`
- `com.kh.trip.controller`
- `com.kh.trip.config`

## 2. 도메인별 구성

### 2.1 user

- domain: `User`, `MemberGrade`, `HostProfile`, `MileageHistory`, `UserAuthProvider`, `UserRole`, `UserRefreshToken`
- repository: `UserRepository`, `MemberGradeRepository`, `UserAuthProviderRepository`, `UserRoleRepository`, `MileageHistoryRepository`, `HostProfileRepository`, `UserRefreshTokenRepository`
- service: `UserService`, `HostService`, `MileageService`
- controller: `UserController`, `HostController`, `MileageController`

### 2.2 login

- service: `LoginService`
- controller: `LoginController`

### 2.3 event

- domain: `Event`, `Coupon`, `EventCoupon`
- service: `EventService`, `CouponService`
- controller: `EventController`, `CouponController`

### 2.4 reserve

- domain: `Booking`, `Payment`, `Review`, `ReviewImage`
- service: `BookingService`, `PaymentService`, `ReviewService`
- controller: `BookingController`, `PaymentController`, `ReviewController`

### 2.5 lodging

- domain: `Lodging`, `Room`, `Wishlist`, `LodgingImage`
- service: `LodgingService`, `RoomService`, `WishlistService`
- controller: `LodgingController`, `RoomController`, `WishlistController`

### 2.6 inquiry

- domain: `InquiryRoom`, `InquiryMessage`
- service: `InquiryService`
- controller: `InquiryController`

### 2.7 admin

- domain: `AuditLog`
- service: `AdminService`
- controller: `AdminController`

## 3. 구조 규칙

- 엔티티는 단수형 사용
- DTO는 `DTO` 접미사 사용
- 서비스 구현체는 `Impl` 접미사 사용
- 오타/중복 명칭은 개정 구조 기준으로 통일

## 4. 상세 기준

- 상세 구조는 `tripzone-structure-spec-v2.md`를 따른다.
