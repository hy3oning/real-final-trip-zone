# TripZone DB 명세서

## 1. DB 설계 원칙

- 기본키는 `NUMBER(10)` + 시퀀스 사용
- 기본 등록/수정일 컬럼은 `REG_DATE`, `UPD_DATE`
- 상태 기반 운영을 우선하고, 필요 시 soft-delete 컬럼 사용

## 2. 주요 테이블

### 2.1 회원 영역

- `MEMBER_GRADES`
- `USERS`
- `USER_AUTH_PROVIDERS`
- `USER_ROLES`
- `USER_REFRESH_TOKENS`
- `HOST_PROFILES`

### 2.2 상품/예약 영역

- `LODGINGS`
- `ROOMS`
- `BOOKINGS`
- `PAYMENTS`
- `LODGING_IMAGES`

### 2.3 리뷰/찜 영역

- `REVIEWS`
- `REVIEW_IMAGES`
- `WISHLISTS`

### 2.4 운영/혜택 영역

- `EVENTS`
- `COUPONS`
- `USER_COUPONS`
- `EVENT_COUPONS`
- `MILEAGE_HISTORY`
- `AUDIT_LOGS`

### 2.5 문의 영역

- `INQUIRY_ROOMS`
- `INQUIRY_MESSAGES`

## 3. 핵심 관계

- `USERS` -> `MEMBER_GRADES`
- `HOST_PROFILES` -> `USERS`
- `LODGINGS` -> `HOST_PROFILES`
- `ROOMS` -> `LODGINGS`
- `BOOKINGS` -> `USERS`, `ROOMS`, `USER_COUPONS`
- `PAYMENTS` -> `BOOKINGS`
- `REVIEWS` -> `BOOKINGS`, `USERS`, `LODGINGS`
- `WISHLISTS` -> `USERS`, `LODGINGS`
- `INQUIRY_ROOMS` -> `USERS`, `HOST_PROFILES`, `LODGINGS`, `BOOKINGS`
- `INQUIRY_MESSAGES` -> `INQUIRY_ROOMS`, `USERS`

## 4. 상태값 기준

### 4.1 회원 상태

- `ACTIVE`
- `DORMANT`
- `BLOCKED`
- `DELETED`

### 4.2 판매자 상태

- `PENDING`
- `APPROVED`
- `REJECTED`
- `SUSPENDED`

### 4.3 예약 상태

- `PENDING`
- `CONFIRMED`
- `CANCELED`
- `COMPLETED`
- `NO_SHOW`

### 4.4 결제 상태

- `READY`
- `PAID`
- `FAILED`
- `CANCELED`
- `PARTIAL_CANCELED`
- `REFUNDED`

### 4.5 문의 상태

- `OPEN`
- `ANSWERED`
- `CLOSED`
- `BLOCKED`

## 5. 정합성 기준

- 이메일 유니크
- 동일 회원/숙소 찜 중복 금지
- 동일 예약 리뷰 중복 금지
- 체크아웃 날짜는 체크인 날짜보다 뒤여야 함
- 환불 금액은 결제 금액 초과 불가

## 6. 상세 기준

- 실행용 DDL은 `tripzone-ddl-v2.sql`을 따른다.
